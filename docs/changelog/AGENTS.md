# Changelog-Anweisungen

Gilt für Änderungen unter `docs/changelog/` zusätzlich zu den Root-Anweisungen.

- `README.md` ist der chronologische Einstiegspunkt; neueste Einträge stehen oben.
- Für jeden tatsächlich nach `main` gemergten Pull Request eine Datei unter `pr/` mit PR-Nummer,
  Datum, Branch, Merge-Commit und kurzer Zusammenfassung pflegen.
- Merge-Metadaten erst eintragen, wenn sie existieren; niemals Nummern oder SHAs erfinden.
- Unter `branches/` die PRs und den Status des zugehörigen Arbeitsbranches nachführen.
- Nach Abschluss oder Merge PR-Datei, Branch-Datei und Übersicht gemeinsam aktualisieren.
- Reine technische Synchronisations-Merges ohne eigene fachliche Änderung erhalten keinen
  duplizierten Feature-Eintrag.
